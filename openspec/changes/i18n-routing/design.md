# Design: i18n-routing

## Architecture

```
src/lib/i18n/
└── index.ts          # Language types, helpers, URL utilities

src/routes/
├── +layout.ts        # Load function: detect lang from URL
├── +layout.svelte    # Hreflang tags, passes lang to children
└── ...
```

## Language detection

URL-based only. The root layout's `+layout.ts` reads `url.pathname`:

- `/` or `/something/` → lang = `'en'`
- `/es/` or `/es/something/` → lang = `'es'`
- `/gl/` or `/gl/something/` → lang = `'gl'`

Available to all pages via `$page.data.lang`.

## Hreflang tags

Every page emits in `<svelte:head>`:

```html
<link rel="alternate" hreflang="en" href="https://anceu.com/the-space/" />
<link rel="alternate" hreflang="es" href="https://anceu.com/es/the-space/" />
<link rel="alternate" hreflang="gl" href="https://anceu.com/gl/the-space/" />
<link rel="alternate" hreflang="x-default" href="https://anceu.com/the-space/" />
```

Generated by `alternateLinks(pathname, PUBLIC_SITE_URL)` in i18n utils.

## Language switcher

In Header, right side of nav. Shows current language and a dropdown/menu to switch.

Links preserve the current slug across languages:
- On `/es/the-space/` → switcher links to `/the-space/` (EN) and `/gl/the-space/` (GL)
- On `/` → switcher links to `/es/` (ES) and `/gl/` (GL)

## Data flow

```
URL request
  → +layout.ts load({ url })
    → returns { lang, pathname }
  → +layout.svelte
    → reads data.lang
    → generates hreflang tags via alternateLinks()
    → renders Header with language switcher
    → renders <slot/>
  → Page reads $page.data.lang for language-aware rendering
```
